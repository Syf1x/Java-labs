# Система менеджмента сотрудников в офисе

Данный проект разработан в рамках учебной дисциплины **«Программирование на языках высокого уровня»**. Приложение представляет собой систему для автоматизации учета сотрудников, управления структурой офиса и отслеживания рабочих процессов.

---

## О проекте
Основная цель системы — упростить взаимодействие и работу с персоналом главам отделов, руководству и HR

---

## Лабораторная работа №1: Базовая архитектура и REST API

В рамках первой лабораторной работы реализован базовый каркас приложения для сущности **Employee (Сотрудник)**.

### Основной функционал:
1. **Слоистая архитектура (Layered Architecture):**
   - `Controller`: Обработка входящих HTTP-запросов.
   - `Service`: Реализация бизнес-логики.
   - `Repository`: Взаимодействие с данными (InMemory/DB).
   
2. **DTO & Mapping:**
   - Внедрена прослойка **DTO (Data Transfer Object)** для разделения внутренней модели данных (Entity) и ответа API.
   - Реализован компонент `Mapper` для преобразования объектов.

3. **REST Endpoints:**
   - `GET /api/employees/{id}` — получение сотрудника по ID с использованием `@PathVariable`.
   - `GET /api/employees/search` — поиск сотрудников по параметрам (например, по фамилии) с использованием `@RequestParam`.

4. **Code Quality:**
   - Настроен плагин **Checkstyle** (Google Java Style) для обеспечения чистоты и единообразия исходного кода.
